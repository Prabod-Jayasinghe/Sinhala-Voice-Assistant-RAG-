**Average confidence:** 0.296 | **Average char overlap:** 46.5%

### Interpretation

- Whisper **correctly detected the language as "Sinhala"** in all 5 cases — it is recognizing the language
- Whisper **is producing Sinhala Unicode script output**, not Latin or gibberish — the model understands the domain
- **Word-level accuracy is low** — transcripts contain the right script/phonetic neighborhood but wrong words
- This is **exactly what the SDLC predicted** (Section 3, NFR): *"Whisper's Sinhala word-error-rate is meaningfully higher than English"* — this is a known, documented limitation of Sinhala as a low-resource language, **not a build error**
- Important note: these round-trip samples used **TTS-generated audio**, which is synthetic/clean speech. Real human speech may perform differently (potentially better or worse)

## Architecture Decision: MARGINAL QUALITY PATH

Per the SDLC (Section 15, risk item 2):
> *"If quality is rough, consider demoing with a curated set of clearly-spoken sample questions rather than fully live, unscripted mic input for interviews."*

**Decision taken:** Proceed with Phase 1 (full architecture) with the following adaptations:

1. **Confidence threshold UI**: Show `stt_confidence` prominently in UI. Below threshold (< 0.4), show "Did I hear that right? [retry]" prompt — as specified in SDLC Section 10 edge case handling.
2. **Text fallback always visible**: FR-8 text input is always on screen, not hidden behind the mic UI. Given STT quality, this fallback is likely to be used frequently.
3. **Curated demo questions**: Provide 5–10 pre-built "try these" Sinhala questions in the UI with clear pronunciation guides — this makes demos reliable regardless of live STT quality.
4. **Honest README**: Document STT accuracy limitation explicitly (Section 11 of SDLC), cite the actual confidence numbers from this Phase 0 test.
